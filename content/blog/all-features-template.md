这篇是"功能大全"模板，专门用来看 Markdown / Hugo 这套骨架支持哪些写法，也可以直接复制某一段去当新文章的起点。**这篇本身应该永远保持 `draft: true`，不建议真的发布**——内容是功能演示，不是真正想给读者看的东西。

---

## 一、文字强调

**加粗** 用 `**加粗**`
*斜体* 用 `*斜体*`
***加粗斜体*** 用 `***加粗斜体***`
~~删除线~~ 用 `~~删除线~~`
`行内代码` 用反引号包起来

---

## 二、标题层级

正文里最高只用到 `##`（两个 `#`），因为文章标题本身已经是 `<h1>` 了（模板自动生成，不用你在正文里再写一个 `#`）。往下每多一个 `#` 缩小一级：

### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题

---

## 三、列表

无序列表：

- 第一项
- 第二项
  - 嵌套的子项
  - 另一个子项
- 第三项

有序列表：

1. 第一步
2. 第二步
3. 第三步

任务清单（打勾的checkbox，GitHub 那种）：

- [x] 已完成的事
- [ ] 还没做的事
- [ ] 另一件还没做的事

## 五、代码块

带语言标注的代码块（会自动高亮，比如这段 Python）：

```python
def hello(name):
    print(f"Hello, {name}!")
```

不标语言，纯文本代码块：

```
就是纯文本，没有语法高亮
```

## 十二、原生 HTML（因为 `hugo.toml` 开了 unsafe = true）

可以直接插普通 markdown 不好表达的东西，比如自定义颜色的文字：

<span style="color:#bc5e3c">这行字是手写 HTML 控制的颜色</span>

也是插视频、iframe 嵌入的地方（写法见编辑指南"公式/图片/视频"那节）。

---

## 十三、这套骨架目前**没有**默认启用的功能（想要的话告诉我，加一行 `hugo.toml` 配置就行）

- 定义列表（术语: 解释 这种格式）
- 上标/下标（比如 x² 这种，不是指数公式里的，是纯文字里的）
- `==高亮标记==` 这种黄色高亮语法
- 目录自动生成（TOC）——现在页面模板没有读取这个的代码
